# Sia CIWS Anti System v1.1.0 🛡️

Sia CIWS Anti System은 실시간으로 네트워크 패킷을 시뮬레이션하고 분석하는 GUI 애플리케이션입니다. Tkinter를 사용하여 제작되었으며, 지능형 분석을 통해 DDoS, 플러딩, 포트 스캔 등 다양한 위협을 탐지하고 자동으로 격리합니다. 또한, 심각한 위협에 대해서는 사용자의 승인 하에 시뮬레이션된 능동적 대응(Counter Strike)을 수행할 수 있습니다.

## ✨ 주요 기능 (v1.1.0)

### 1. 실시간 패킷 캡처 및 분석
- **실시간 시뮬레이션**: '캡처 시작/중지' 버튼으로 실시간 패킷 데이터 생성을 제어합니다.
- **위험도 시각화**: 패킷의 크기에 따라 위험도를 4단계(`low`, `medium`, `high`, `critical`)로 나누고, 각기 다른 색상으로 목록에 표시하여 직관성을 높였습니다.
- **상세 정보 조회**: 목록에서 특정 패킷을 선택하면 하단에 출발지/목적지 IP, 포트, 패킷 길이 등 상세 정보가 표시됩니다.
- **위험도 필터링**: '위험도 필터' 버튼을 통해 보고 싶은 위험도 수준의 패킷만 필터링하여 볼 수 있습니다.

### 2. 지능형 CIWS 자동 방어 시스템
`CIWS 활성화` 시, 단순 패킷 크기뿐만 아니라 트래픽 패턴과 빈도를 분석하여 복합적인 위협을 자동으로 탐지하고 격리합니다.

- **대용량 패킷 차단**: 설정된 임계값(`2000` 바이트) 이상의 비정상적인 패킷을 즉시 격리합니다.
- **DDoS 공격 의심 탐지**: 짧은 시간(`2.0`초) 내에 비정상적으로 많은 패킷(`15`개 이상)이 집중될 경우, 이를 고빈도 공격으로 간주하여 관련 패킷을 격리합니다.
- **플러딩(Flooding) 공격 탐지**: 특정 IP에서 비정상적으로 많은 패킷(`3.0`초 내 `10`개 이상)이 반복 감지될 경우, '플러딩 공격'으로 탐지하고 해당 패킷을 격리합니다.
- **비정상 포트 스캔 탐지**: 알려진 포트 외의 비정상 포트로 접근하는 패킷 비율이 임계치(`30%`)를 초과하면 경고를 발생시키고 관련 패킷을 격리합니다.

### 3. 능동적 위협 대응 (Counter Strike)
CIWS가 `high` 또는 `critical` 등급의 심각한 위협을 격리했을 때, 사용자가 직접 공격 소스로 시뮬레이션된 역공격을 실행할 수 있는 기능입니다.

- **사용자 승인 기반 역공격**: '격리' 목록에서 위험 등급이 높은 위협을 선택하면 `⚡ Counter Strike` 버튼이 활성화됩니다.
- **대응 시뮬레이션**: 사용자가 역공격을 승인하면, 애플리케이션은 공격자 IP를 대상으로 가상의 대응 패킷을 전송하는 과정을 시뮬레이션하고 로그를 통해 피드백을 제공합니다.
- **시각적 피드백**: UI의 'Counter Strike 로그' 창을 통해 역추적, 대응 패킷 생성, 전송 등 작전 진행 상황을 실시간으로 확인할 수 있습니다.

### 4. 데이터 관리 및 리포트
- **목록 지우기**: 현재까지 수집된 모든 패킷 및 분석 데이터를 초기화합니다.
- **캡처 다운로드**: 현재까지 수집된 모든 통계, 공격 로그, 패킷 정보를 `captured_packets.txt` 파일로 저장하여 리포트를 생성합니다.

## 🚀 실행 방법

1. Python 3.x 버전이 설치되어 있는지 확인합니다.
2. 터미널에서 아래 명령어를 실행하여 애플리케이션을 시작합니다.

```bash
python AntiSystem.py
```

### ✅ 버전 1.1.0 (현재)
- **능동적 위협 대응 (Counter Strike) 기능 추가**: `high`, `critical` 등급 위협에 대한 시뮬레이션 역공격 기능.
- **지능형 CIWS 방어 시스템 도입**:
  - 대용량 패킷 차단 (임계값: `2000` 바이트)
  - DDoS 공격 의심 탐지 (패턴 및 빈도 기반)
  - 플러딩 공격 탐지 (단일 IP 집중 트래픽 기반)
  - 비정상 포트 스캔 탐지 (포트 접근 비율 기반)

### ✅ 버전 1.0.0 (완료)
- 실시간 패킷 캡처 시뮬레이션 및 GUI 표시
- 패킷 크기(Length) 기반 위험도 분석 및 색상 분류
- 기본 CIWS 모드: `4000`바이트 이상의 패킷을 '격리' 목록으로 이동
- 위험도 수준별 필터링 기능
- 캡처된 패킷 로그 파일 다운로드 기능

---

> **⚠️ 주의사항**
>
> `Counter Strike` 기능은 애플리케이션 내에서의 **교육 및 시뮬레이션 목적**으로만 구현되었습니다. 실제 네트워크 환경에서 허가받지 않은 역공격은 불법이 될 수 있으며, 사이버 분쟁을 심화시킬 수 있습니다. 본 기능은 '능동적 방어'의 개념을 학습하고 이해하는 데 목적을 둡니다.
